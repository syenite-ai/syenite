# Syenite — Revenue Model

---

## Pricing Principle

**Interest spread, not fee stacking.** Syenite captures the delta between protocol rates and client rates — through rebalancing alpha, avoided liquidation penalties, and routing optimization. No AUC fees on principal. The institution sees two paths: direct (deploy into Morpho/Aave, manage your own position, get raw protocol yield) or managed (deploy through Syenite, get optimized lending with protection, pay a spread on interest). The spread is justified by the service. If Syenite doesn't improve the outcome, the institution uses the direct path.

Price against the client's alternative:

- **No liquidation protection** → 5-10% loss when protocol liquidation fires. Syenite's spread is insurance that pays for itself on the first auto-unwind event.
- **Sitting on one protocol** → missing 50-150bp of yield spread. Syenite's performance fee only applies when rebalancing captures that spread.
- **Managing DeFi positions manually** → $200-400K/yr in internal ops staffing. Syenite's MLISA replaces that headcount.

## LISA Revenue Streams

### 1. Protection Premium
Annual fee for active auto-unwind coverage on the vault.

| Metric | Value |
|---|---|
| Basis | AUC under active protection |
| Rate | 25-50bp/yr |
| At $50M AUC | $125-250K/yr |
| At $100M AUC | $250-500K/yr |

Collected continuously (per-block or per-epoch). Client pays for the guarantee that the keeper agent is monitoring and will fire auto-unwind before protocol liquidation.

### 2. Routing Spread
Captured on the initial borrow flow when the vault routes to the best protocol.

| Metric | Value |
|---|---|
| Basis | Borrow volume |
| Rate | 5-10bp per borrow |
| Assumption | 65% average LTV, ~2x annual turnover |
| At $50M AUC | $32-65K/yr |
| At $100M AUC | $65-130K/yr |

### 3. Performance Fee on Rebalancing
When cross-protocol migration improves the client's rate, Syenite takes a share of the incremental yield.

| Metric | Value |
|---|---|
| Basis | Incremental yield captured through rebalancing |
| Rate | 15-20% of alpha |
| Assumption | Average 50bp rate improvement, 30% of AUC rebalanced annually |
| At $50M AUC | ~$11-15K/yr |
| At $100M AUC | ~$22-30K/yr |

Variable. Depends on rate environment and volatility. Higher in volatile markets (more rebalancing opportunities).

### 4. Auto-Unwind Event Fee
Charged when auto-unwind fires. The client pays for the execution that saved them from a much larger liquidation penalty.

| Metric | Value |
|---|---|
| Basis | Position size at time of auto-unwind |
| Rate | 0.3-0.5% of position |
| Assumption | 5-10% of positions trigger auto-unwind annually |
| At $50M AUC | ~$7.5-25K/yr |
| At $100M AUC | ~$15-50K/yr |

Highly variable. In a calm market, this is zero. In a volatile quarter, this can exceed all other revenue streams. A single auto-unwind on a $5M position at 0.5% = $25K.

## MCP Lending Server Revenue

The MCP server is a distribution channel, not a revenue center. Revenue comes from the vault infrastructure underneath — agents that open positions through the MCP server create Syenite vaults and generate interest spread revenue.

### Open Core Model

| Tier | Tools | Price | Purpose |
|---|---|---|---|
| Free | rates.query, position.monitor, risk.assess, market.overview | $0 | Adoption layer — best DeFi lending data source for agents |
| Paid (execution) | position.open, position.unwind, position.rebalance, vault.create | Vault-level interest spread | Revenue layer — vault creation drives spread revenue |

### Why Free Tier Matters

The DeFi MCP niche has ~27 servers (March 2026). Zero target institutional lending. The free tier captures the category before competition materializes. Agent developers integrate the read tools because they're genuinely useful. When their agents graduate from read to write, the Syenite vault is the obvious next step.

85M+ weekly MCP SDK downloads across 33K+ dependents. All major agent frameworks (CrewAI, LangChain, OpenAI Agents SDK) support MCP natively. Cross-vendor client support (Claude, ChatGPT, Cursor, VS Code, Windsurf, Cline). The addressable developer audience is large and growing.

### Revenue Mechanics

1. Agent calls `lending.position.open` via MCP → Syenite vault is created (Safe proxy)
2. Vault executes lending position on Aave/Morpho with auto-unwind configured
3. Syenite captures interest spread on the managed position
4. Agent monitors via `lending.position.monitor` (free)
5. Agent adjusts/rebalances via paid tools → vault activity continues → spread revenue continues

The MCP server generates the same vault-level revenue as institutional clients. An agent managing a $5M BTC lending position produces the same spread as an institution managing $5M through the AC SA adaptor. The distribution channel is different; the revenue model is identical.

### Agent Volume Thesis

Human traders interact with DeFi once or twice a day. Agents interact every block. The frequency and volume of agent-driven lending operations will dwarf human operations. An MCP server that becomes the canonical lending interface for agents captures volume that scales with agent adoption, not headcount.

### AC SA + MCP Compound

Institutions using AC Smart Accounts can designate an AI agent as their SA operator. The agent operates tBTC lending positions through the MCP interface, constrained by the SA whitelist (Threshold's gate), the Syenite vault parameters (risk framework), and the protocol's own constraints. Three layers of protection, each enforced by a different party, none of which the agent can override. This is an institutional compliance story for agentic DeFi that doesn't exist anywhere else.
