# Docs (public)

This folder contains **product and architecture** documentation only. Everything here is safe for the public repo.

**In this repo you will find:**
- **architecture/** — Product overview, system layers, vault design. Pure product and technical description.

**Standard policy:** Strategy, commercial, competitive, planning, research, and operational runbooks do **not** live in the public repo. They are kept in `docs/internal/`, which is listed in `.gitignore`. Only pure product information (what we build, how it works, how to use it) is committed. Do not add internal or strategic material to the repo without moving it under `docs/internal/` and ensuring that path remains gitignored.
