Transaction trust layer

Three tools sit between a constructed transaction and a signature: tx.verify (who you are calling), tx.simulate (whether it succeeds on current state), tx.guard (whether it passes your policy). They work on any unsigned transaction, including calldata from swap.quote or another MCP.

tx.verify

Checks the to address: contract vs EOA, Etherscan verification, Sourcify match, Syenite protocol registry, and optional function selector decode via 4byte.directory.

Parameters: to (address), chain (ethereum, arbitrum, base, bsc), optional data (calldata for selector decode).

Response highlights: verification.etherscan, verification.sourcify, protocol.syeniteAllowlisted, riskFlags (e.g. unverified_contract, unknown_protocol, proxy_contract, eoa_target), summary.

tx.simulate

Runs eth_call at the current block with the proposed tx. Reports success or revert, gas estimate, native value outflow from from, and approximate gas cost in USD. Reverts mapped to token_approval_required when the error suggests allowance issues.

Parameters: transaction object: to, data, from, optional value, optional chainId; optional top-level chain (name or id).

Chains: ethereum, arbitrum, base, bsc (same as wallet and gas tools).

Limits: full ERC-20 balance deltas need trace-level simulation (e.g. Tenderly); the tool documents this in its note field. Results are verifiable by repeating the same eth_call against any RPC for that block.

tx.guard

Evaluates transaction against rules you supply. Syenite does not change your rules.

Rules (all optional except you must pass at least one): maxValueNative (string, ETH-equivalent cap), allowedContracts, blockedContracts, allowedFunctions (selectors or hex strings), requireVerifiedContract (boolean: emits a skip telling you to run tx.verify first), requireAllowlisted (must match Syenite curated registry), maxGasLimit (with transaction.gasLimit set).

Response: checks with pass / fail / skip, approved when there are no failures and no skips, summary.

Recommended order

1. swap.quote, lending.supply, or other source returns transactionRequest. 2. tx.verify on to and calldata. 3. tx.simulate with full tx. 4. tx.guard with your caps and allowlists. 5. Sign only if verify + simulate + guard outcomes are acceptable. 6. After signing, call tx.receipt to confirm the transaction landed and decode its events.
